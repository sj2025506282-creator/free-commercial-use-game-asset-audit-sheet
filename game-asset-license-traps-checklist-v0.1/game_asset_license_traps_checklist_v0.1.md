# 7 Game Asset License Traps To Check Before Shipping

This checklist is for indie developers, solo devs, and game-jam teams using free or low-cost game assets.

It is not legal advice. It is a practical review pass for spotting obvious risks and missing evidence.

## Trap 1: Treating "Free" As One Safe Bucket

Check whether the asset is:

- CC0 / public domain
- CC BY / attribution required
- free for personal use only
- free for commercial use
- free to use in finished projects but not to redistribute as raw files
- under a custom license
- unclear

Action: record the exact visible license signal, not just "free".

## Trap 2: Mixing CC0 And Attribution-Required Assets

CC0 assets usually do not require attribution. CC BY and many custom licenses do.

Action: keep attribution-required assets in a separate tracker and prepare credits before shipping.

## Trap 3: Trusting Directory Tags More Than Source Pages

An asset directory may tag something as CC0 or commercial-use, but the creator/source page may be different, missing, or outdated.

Action: save the original source page and visible license evidence.

## Trap 4: Forgetting Font Licensing

Fonts can have separate rules for:

- embedding in a game/app
- modification
- logo or title use
- redistribution
- web vs desktop vs app use
- editorial vs commercial use

Action: review fonts separately from art/audio assets.

## Trap 5: Redistributing Raw Asset Files

Some licenses allow use in a finished game but do not allow repackaging or redistributing the raw asset files.

Action: do not include third-party raw assets in your own asset pack unless the license clearly allows redistribution.

## Trap 6: Losing Source And Provenance Notes

If you cannot remember where a file came from, you cannot re-check its license before release.

Action: keep source URL, download date, license signal, and notes in a provenance log.

## Trap 7: Skipping A Final Pre-Release Re-Check

License pages can change, links can break, and your project scope can change from prototype to commercial release.

Action: re-check important asset sources before public/commercial release.

## Final Questions

- Do all third-party assets have source URLs?
- Are attribution-required assets separated from CC0 assets?
- Are all fonts checked separately?
- Are raw third-party assets excluded from redistributed packs?
- Are unclear, NC, ND, or custom-license assets removed or flagged?
- Has the release build been reviewed close to shipping date?
